# deplambda Rule Port Validation

This directory contains test infrastructure for validating CHISEL's port of deplambda rules from Reddy et al. (2016, TACL).

## Overview

CHISEL implements deplambda's linguistic rules in three components:
1. **Tree transformations** (chisel/tree_transformations.py): Restructure dependency tree
2. **Templates** (chisel/deplambda_templates.yaml): Pattern matching for composition
3. **Constructors** (chisel/deplambda_constructors.py): Lambda expression builders

To verify correctness, we compare CHISEL's output against gold-standard logical forms generated by the original UDepLambda Java implementation.

## Test Sentences

`deplambda_test_sentences.txt` contains 50 sentences covering all linguistic constructions:
- Simple declaratives (5)
- Wh-questions (10): who, what, where, when, how many, which, whose, why, how
- Relative clauses (8): who, which, that, whose, where, when
- Passives (5)
- Comparatives (4)
- Superlatives (4)
- Coordination (4): and, or, but
- Negation (3)
- Existentials (2)
- Prepositional phrases (3)
- Complex multi-clause (2)

## Generating Gold Standard with UDepLambda

### Prerequisites

- Java 8+ (tested with OpenJDK 17)
- Apache Ant

### Setup

```bash
# Clone UDepLambda
git clone https://github.com/sivareddyg/UDepLambda.git
cd UDepLambda

# Initialize submodules
git submodule update --init --recursive lib
git submodule update --init --recursive lib_data/ud-models-v1.3

# Build
ant build
```

### Generate Logical Forms

```bash
# Convert test sentences to JSON format
python3 << 'EOF'
import json

with open('../chisel/tests/deplambda_test_sentences.txt', 'r') as f:
    lines = [line.strip() for line in f if line.strip() and not line.startswith('#')]

with open('test-input.json', 'w') as f:
    for line in lines:
        json.dump({'sentence': line}, f)
        f.write('\n')
EOF

# Run UDepLambda English pipeline
./run.sh en test-input.json test-output.json

# Check debug output for step-by-step derivations
cat debug.txt
```

### Extract Gold Data

```bash
# Parse UDepLambda output and debug logs
python3 << 'EOF'
import json

# Read output
with open('test-output.json', 'r') as f:
    outputs = [json.loads(line) for line in f]

# Read debug log for transformation steps
with open('debug.txt', 'r') as f:
    debug = f.read()

# TODO: Parse debug.txt to extract transformation and assignment steps
# Format: tests/deplambda_gold.json

gold_data = []
for output in outputs:
    gold_data.append({
        'sentence': output['sentence'],
        'ud_parse': output.get('ud_parse', {}),
        'transformation_steps': [],  # Extract from debug.txt
        'assignment_steps': [],      # Extract from debug.txt
        'final_logical_form': output.get('logical_form', ''),
        'notes': ''
    })

with open('../chisel/tests/deplambda_gold.json', 'w') as f:
    json.dump(gold_data, f, indent=2)
EOF
```

### Cleanup

After generating `tests/deplambda_gold.json`:

```bash
# Remove UDepLambda repo (no runtime dependency)
cd ..
rm -rf UDepLambda
```

## Running Tests

```bash
# After generating gold data
cd chisel
pytest tests/test_deplambda_port.py -v
```

## Test Structure

Each test case verifies:
1. **Transformation correctness**: Tree structure after transformations matches expected
2. **Assignment correctness**: Lambda expressions assigned to each node match expected
3. **Alpha-equivalence**: Final logical form is alpha-equivalent to gold standard

Tests use `nltk.sem.logic` for alpha-equivalence checking (same structure, variable names may differ).
